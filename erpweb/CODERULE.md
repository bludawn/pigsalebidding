# 后台开发规范圣经

### 1. 业务表sql建表规范
  - 表名必须用t_ 开头，名称用驼峰命名，如t_pigResource。
  - 字段名必须用f_ 开头，名称用驼峰命名，如f_id，f_name，f_orderId，f_customerPhoneNumber。总长度不能超过32位，超过32个考虑缩写。
  - 每个主表都是f_id字段，分录表必须有f_entryId和f_id字段
  - 每个主表必须有审计字段 f_createTime，f_updateTime，f_creator，f_updator
  - 业务上有关联，但表中不要建外键
  
### 2. web生猪业务页面，所有需要输入id的输入框，都是打开对应列表选择。
  - 列表查看时，都是显示对应数据的名称
  - 修改页面选择后，反显也是显示名称，但保存时是保存id
  - 数组的话，则是多选。
  - 非数组的话，则是单选。
  - 如果没有名称，则显示编码。
  - 如果是收货、送货地址，则显示 通过地址code 找到地区 + 详细地址。

### 3. 所有生猪业务数据列表，都能通过 名称 或 编号 进入 数据查看态 页面，此页面无法编辑，名称 或 编号 变超链接蓝字（无下划线）。

### 4. web生猪业务页面，所有需要输入枚举值的输入框，都是打开对应枚举值下拉框选择。
  - 列表查看时，都是显示对应枚举值的名称
  - 修改页面选择后，反显也是显示名称，但保存时是保存枚举值

### 5. 列表能显示所有字段，包括审计字段。如果是图片则显示图片，如果是视频，则显示视频第一帧，如果是图片和视频数组，则 显示第一个的图片或视频第一帧。
